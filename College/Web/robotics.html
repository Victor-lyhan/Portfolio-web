Software & Controls

Field-Centric Drive

Click for Description

Allows easier driving by transforming joystick inputs by robot heading: x' = x cosθ + y sinθ, y' = −x sinθ + y cosθ, keeping driver controls intuitive at any orientation.

PIDF & Motion Profiles

Click for Description

Segmented PIDF + trapezoidal profiles limit accel/vel for smooth, repeatable lift & arm trajectories; tuned per segment.

Camera-Assisted Auto

Click for Description

Vision tags samples, planner selects the nearest legal target, then executes a pose-corrected trajectory for consistent auto points.

Strategy & Design History Framework (DHF)

We followed a lightweight DHF discipline—Requirements(Gamepoint matrix) → Risk (FMEA) → Controls/Design Decisions → Verification(Monte Carlo)—so every subsystem change tied back to match value and reliability. This professional flow let us debate ideas with data, not opinion.

DHF (Process)

Single source of truth for requirements, trades, and rationale; kept judges and alliance partners on the same page.

FMEA (Risk)

Top failure modes—slip, stall, mis-align—drove interlocks, current limits, and recovery routines.

Monte Carlo (Consistency)

Simulated cycle variance & traffic to plan conservative, repeatable match scripts.
